# 截面因子与跨标的数据对齐 — 讨论记录

> **日期**：2026-03-14
> **参与方**：项目负责人 + Claude Code（架构审查）
> **背景**：Phase 2a 因子研究模块设计讨论中，针对截面因子的适用性和 tick 数据非同步对齐问题的深入讨论

---

## 一、截面因子在 5 标的场景的适用性

### 问题

传统股票截面分析动辄几百上千只标的，本项目仅 5 个 crypto 标的（BTC/ETH/SOL/BNB/DOGE），截面分层回测的统计意义有限。

### 结论

- **框架设计上必须完整支持截面分析**（扩展性要求 + 面试展示）
- **实际研究中以时序因子 + 时序信号为主**，截面能力在标的扩展到 20-50 个时直接可用
- 5 标的场景下，截面分层可分为 5 组（每组 1 个），虽然统计意义弱，但仍能看到因子的排序能力

## 二、因子的三种类型

讨论过程中确认了因子的分类需要支持三种类型：

| 类型 | 输入 | 输出 | 举例 |
|------|------|------|------|
| **时序因子** | 单标的数据 | 该标的的因子值 | BTC 5min 波动率 |
| **截面因子** | 所有标的数据 | 各标的的排名/标准化值 | 5 币动量排名 |
| **跨标的因子** | 指定标的（如 BTC）的数据 | 其他标的的因子值 | BTC 涨跌 → ETH 领先信号 |

跨标的因子是讨论中新增的第三类。在 crypto 市场中，BTC 的主导地位使得跨标的效应尤为显著：

- **领先-滞后**：BTC 短期收益 → 预测 ETH/SOL 等的后续走势
- **波动率传染**：BTC 波动率突增 → 其他币种波动率跟随
- **订单簿压力溢出**：BTC 订单簿极度不平衡 → 其他币种跟随反应
- **相关性偏离**：BTC-ETH 滚动相关性偏离长期均值 → 均值回复信号

## 三、逐笔成交数据的非同步交易问题

### 3.1 问题描述

进行截面分析时需要比较不同标的在"同一时刻"的因子值。但逐笔成交数据是不规则时间序列——当 BTC 发生一笔交易时，ETH 在物理上的同一时刻可能并没有交易发生。

核心困难：**价格只在成交发生时才被"定义"**。两笔成交之间，资产的"真实价格"是一个不可观测的潜变量。

### 3.2 工程方案（默认使用）

**分层对齐**：不在原始 tick 层面做截面对齐，而是让每个标的独立消耗全量 tick 数据计算因子值，在因子输出层面再做对齐。

```
原始 tick 数据（各标的独立，不规则时间）
    │
    ▼  每个标的独立计算时序因子（利用全部 tick，无损失）
因子值序列（由因子声明输出频率，如 "1s"）
    │
    ▼  在因子值层面进行网格对齐（forward fill）
因子面板（统一时间索引）
```

**优势**：
- 原始 tick 数据全量用于因子计算，零损失
- 对齐误差被限制在因子输出频率的精度内（如 1 秒级）
- 对分钟级及以上策略，误差可忽略

**局限**：
- 在因子值层面仍存在"物理不同时"的问题
- 理论上不够严格

### 3.3 学术严格方案

#### 方案 1：刷新时间采样（Refresh Time Sampling）

定义"刷新时刻"为**所有标的自上一个刷新时刻以来都至少发生过一笔新成交**的时刻：

```
BTC ticks:  --|---|-|-------|--|--->
ETH ticks:  ----|--|---|----|---|--->
SOL ticks:  --|----|---|--|----|---->
                      ↑       ↑
                   刷新时刻  刷新时刻
              （三者都有了新成交）
```

在刷新时刻，每个标的取其最近一笔成交。这样保证所有标的的数据都是"新鲜的"。

**代价**：刷新频率取决于最不活跃的标的。如果 DOGE 10 秒才成交一笔，刷新频率不会高于 0.1Hz。

#### 方案 2：Hayashi-Yoshida 估计量

专门解决非同步交易下的**协方差/相关性**估计。核心思想：不要求时间对齐，而是用所有时间区间有重叠的成交对来计算协方差。

```
BTC: |--Δp₁--|---Δp₂---|
ETH:    |---Δq₁---|--Δq₂--|

只要 BTC 的 Δpᵢ 和 ETH 的 Δqⱼ 在时间上有重叠，
就把 Δpᵢ × Δqⱼ 计入协方差估计
```

避免了传统同步采样导致的 **Epps 效应**（采样频率越高，估计的相关性越低的假象）。

**适用场景**：计算因子间相关性、标的间协方差时使用。不直接用于因子值对齐。

#### 方案 3：潜在价格模型

假设每个资产存在连续的潜在有效价格过程 p*(t)，观测到的成交价是 p*(t) 加上微观结构噪声。通过状态空间模型（如卡尔曼滤波）估计 p*(t)，在 p*(t) 层面做对齐。

理论上最优雅，但实现复杂且对模型假设敏感。本项目不采用。

### 3.4 本项目的实际处理方案

| 场景 | 方案 | 理由 |
|------|------|------|
| 时序因子计算 | 无需对齐 | 各标的独立计算 |
| 因子面板构建 | 网格对齐 (forward fill) | 因子输出层面对齐，误差被聚合吸收 |
| 需要严格截面对齐的因子 | 可选：refresh time 采样 | 在因子声明中指定 `alignment="refresh_time"` |
| 计算因子相关性矩阵 | 可选：Hayashi-Yoshida | 避免 Epps 效应 |

### 3.5 面试回答框架

若面试官问"多标的 tick 数据如何对齐"：

1. **承认问题**：非同步交易导致的时间对齐问题是经典的市场微观结构问题
2. **工程方案**：各标的独立计算因子 → 因子值层面网格对齐。优点：零数据损失；缺点：理论不严格
3. **严格方案**：刷新时间采样保证数据新鲜度；Hayashi-Yoshida 估计量避免协方差的 Epps 效应
4. **回到实际**：对于分钟级策略，因子计算的 lookback 窗口（几十秒到几分钟）本身是时间聚合，tick 级非同步误差被聚合吸收，影响可忽略

## 四、多空组合 ≠ 对冲

讨论中确认：在 5 个 crypto 标的中"做多排名前 2、做空排名后 2"是**多空组合**，不是**对冲组合**。

严格的对冲需要：
- **Beta 中性**：组合对 BTC 的 beta = 0
- **Dollar 中性**：多空等金额
- **波动率目标**：总组合波动率控制在目标水平

这些是组合优化层的内容，将在 Phase 2b 中设计。

## 五、非线性因子的评价

IC（Spearman 相关系数）只能捕捉**单调关系**。部分因子（如波动率）与收益存在 U 形或其他非线性关系。

### 解决方案

**评价层面（发现非线性）：**
- 分层收益图：按因子值分 5-10 组，各组平均收益。U 形关系一目了然
- 条件 IC：分段计算 IC（低/中/高三段），符号不同说明非线性
- 互信息（Mutual Information）：捕捉任意统计依赖，不限于单调
- 因子特征曲线：LOWESS 非参数回归拟合 factor_value → return

**建模层面（利用非线性）：**
- 树模型（LightGBM/XGBoost）天然处理非线性
- 因子变换（取绝对值、分桶离散化）作为新因子
- 这是 Phase 2b 模型策略模块的内容
