# 组合优化完整知识体系

本文档系统讲解生产级量化基金的组合优化知识，面向具有单资产量化经验、
缺乏多资产组合优化经验的开发者。目标是建立对完整做法的纵观理解，
并标注本项目所做的简化在行业全景中的位置。

---

## 一、组合优化的核心问题

### 1.1 单期均值-方差（Markowitz）

组合优化的起点是 Markowitz (1952) 的均值-方差框架：

```
maximize:  μ'w - λ × w'Σw
subject to: Σw_i = 1, w_i ≥ 0
```

其中 `μ` 是预期收益向量，`Σ` 是协方差矩阵，`λ` 是风险厌恶系数。

**为什么是起点**：它把投资决策从"凭感觉"变成了一个有数学最优解的凸优化问题。
所有后续方法都是在这个框架上的扩展。

**为什么不够**：
1. **估计误差放大**：优化器会在 `μ` 和 `Σ` 估计误差最大的方向上集中仓位（estimation error maximizer）
2. **忽略交易成本**：从 A 调整到 B 的成本可能吃掉理论收益
3. **单期近视**：不考虑"今天的交易如何影响明天的最优决策"

### 1.2 目标函数三大项

生产级目标函数的完整形式：

```
minimize:  w'Σw                    ← 风险
         - λ × α'w                 ← 预期收益（取负是因为 minimize）
         + cost(Δw, market_state)  ← 交易成本
```

- **风险项** `w'Σw`：组合方差。希望风险越小越好。
- **收益项** `α'w`：alpha 信号与权重的内积。希望在高 alpha 方向上多配。
- **成本项** `cost(Δw)`：权重调整带来的交易成本。希望不必要的换手越少越好。

三者的权衡是组合优化的核心决策。

> **本项目对应**：Phase 2b `PortfolioConstructor` 使用固定 `γ‖Δw‖₁` 作为成本项（线性惩罚）；
> Phase 2c `ExecutionOptimizer` 使用动态 `cost(Δw, MarketContext)` 包含 commission + spread + impact。

### 1.3 约束的本质

约束是业务需求的数学翻译。交易员说"单标的不超过 40%"，数学表达为 `|w_i| ≤ 0.4`。
约束不改变目标函数的方向，而是划定可行域的边界。

## 二、风险模型

### 2.1 从样本协方差到因子风险模型

**样本协方差**：直接用历史收益率计算 `Σ = (1/T) × R'R`。
问题是当 N（标的数）接近 T（样本数）时，估计极不稳定。

**Ledoit-Wolf 收缩**：`Σ_shrunk = δ×F + (1-δ)×S`，
其中 F 是结构化目标（对角矩阵），S 是样本协方差，δ 是最优收缩强度。
这是"用偏差换方差"的经典 bias-variance tradeoff。

> **本项目使用 Ledoit-Wolf**（`alpha_model/portfolio/covariance.py`），
> 因为标的数 N=5，T/N 比足够大，Ledoit-Wolf 已经够用。

**因子风险模型（Barra/Axioma）**：

```
Σ = B × F × B' + D
```

- `B` (N×K)：因子暴露矩阵（N 标的对 K 因子的暴露度）
- `F` (K×K)：因子协方差矩阵（只估计 K 个因子间的协方差）
- `D` (N×N)：对角矩阵，特质风险（每个标的独有的非因子波动）

**为什么因子模型更好**：
- 只需估计 K×K（K 通常 10~50）个协方差参数，而非 N×N（N 可能 500+）
- 经济学直觉更强：风险来源于有限个因子（市场、行业、动量、规模等）
- 可以做风险归因：组合波动的 30% 来自市场因子，20% 来自行业因子……

**何时需要切换**：当标的数 N > 20 时，Ledoit-Wolf 的估计质量开始下降，
应考虑因子风险模型。

### 2.2 风险归因

有了因子风险模型，可以将组合总风险分解为各因子的贡献：

```
Portfolio Variance = w'Σw = w'BFB'w + w'Dw
                   = (因子风险) + (特质风险)
```

进一步可以分解到单个因子。这对于理解"风险从哪来"至关重要。

## 三、交易成本模型

### 3.1 成本分解

实际交易的总成本包含三个分量：

| 分量 | 定义 | 特点 |
|------|------|------|
| **佣金** | 交易所收取的固定比例手续费 | 线性，与交易量成正比 |
| **买卖价差** | (ask - bid) / 2，市场做市商的报酬 | 线性，流动性好则价差小 |
| **市场冲击** | 自身交易推动价格的成本 | 非线性，与交易量的 ~1.5 次幂成正比 |

佣金和价差是"确定性成本"，市场冲击是"概率性成本"，也是最复杂的部分。

### 3.2 市场冲击的经验模型

**线性模型**：`Impact = η × |ΔV|`。简单但不真实——大单的冲击显著超线性。

**Square-root 模型（Almgren-Chriss）**：

```
Impact = η × σ × |ΔV|^0.5 / √ADV
```

这是行业最广泛使用的经验公式。直觉：
- `σ`（波动率）越大，价格越容易被推动
- `|ΔV|`（交易金额）越大，冲击越大
- `ADV`（日均成交量）越大，市场越能消化大单

**幂律模型**：`Impact ∝ |ΔV|^p`，其中 p 通常在 0.5~0.7 之间。

> **本项目使用 1.5 次幂模型**（`execution_optimizer/cost.py`），
> 即 `eff_coeff × |Δw|^1.5`，这是权重空间下的等价表达。
> Phase 2b 的向量化回测使用更简单的 `σ × √(|ΔV|/ADV)` 近似。

### 3.3 成本模型的校准

生产环境中，`η`（冲击系数）不是拍脑袋取值，而是从实际成交数据回归得到：

1. 收集历史订单：下单量、执行均价、到达价格
2. 计算实际冲击：(执行均价 - 到达价) / 到达价
3. 回归模型：`realized_impact ~ η × σ × (V/ADV)^p`

校准频率通常是每月或每季度。不同标的、不同时段的 η 可能差异很大。

> **本项目**使用默认 `impact_coeff=0.1`，支持逐标的 `pd.Series` 形式传入。
> 实盘时应根据实际成交数据校准。

### 3.4 时变性

交易成本不是常数：
- **时段效应**：开盘/收盘流动性好，中间可能较差；加密市场 7×24 但也有流动性波动
- **事件驱动**：重大新闻发布前后，价差急剧扩大，冲击成本飙升
- **自适应**：合理的做法是用滚动窗口或指数加权更新成本参数

> **本项目**通过 `MarketContext` 在每个时间步传入实时的 spread、volatility、ADV，
> 天然支持时变成本。

## 四、约束工程

### 4.1 基本约束

| 约束 | 数学表达 | 业务含义 |
|------|---------|---------|
| 仓位上限 | `\|w_i\| ≤ max_weight` | 防止单标的过度集中 |
| 杠杆上限 | `Σ\|w_i\| ≤ L` | 控制总敞口 |
| 换手率上限 | `Σ\|Δw_i\| ≤ τ` | 限制单步调仓幅度 |
| Dollar-neutral | `Σw_i = 0` | 多空等金额，对冲市场方向 |

### 4.2 风险约束

| 约束 | 数学表达 | 用途 |
|------|---------|------|
| Beta-neutral | `β'w = 0` | 对冲市场系统性风险 |
| 因子暴露上下界 | `l_k ≤ B_k'w ≤ u_k` | 控制对特定因子的暴露 |
| 跟踪误差上界 | `(w-w_bench)'Σ(w-w_bench) ≤ TE²` | 相对基准偏离不超过阈值 |

> **本项目实现了前两层**（仓位上限、Dollar/Beta-neutral），
> 因子暴露约束和跟踪误差约束是大型基金的标配，对 5 标的组合尚不必要。

### 4.3 流动性约束

```
|Δw_i| × V ≤ participation_rate × ADV_i
```

单步交易量不超过 ADV 的 X%。这是防止"自己把自己的价格打飞"的关键约束。

> **本项目在 `ExecutionOptimizer` 中实现**，默认 `max_participation=5%`。

### 4.4 整数约束

"要么至少买 2%，要么不买"→ 混合整数二次规划（MIQP）。
问题从多项式复杂度变为 NP-hard，求解时间可能从毫秒变为分钟。

行业做法：
- 先用连续松弛求解，再后处理（四舍五入到最小手数）
- 或直接用 Gurobi 等支持 MIQP 的商业求解器

> **本项目使用连续权重近似**，对小资金+少标的场景足够。

## 五、多期优化与最优执行

### 5.1 单期 vs 多期

单期优化假设"当前步调到目标权重，一步到位"。但实际上：
- 大单一步执行，冲击巨大
- 分 N 步执行，冲击更小，但信号可能衰减
- 这是**执行速度 vs 冲击成本**的权衡

### 5.2 Almgren-Chriss 最优执行

经典的多期执行框架：给定总交易量 X，在 T 个时间步内完成，如何分配每步的量？

```
minimize:  E[cost] + λ × Var[cost]
```

关键结论：
- 如果冲击是永久的（permanent），最优策略是均匀执行（TWAP）
- 如果冲击是暂时的（temporary），最优策略是前重后轻（aggressive start）
- `λ`（风险厌恶）越大，越倾向于快速执行以减少不确定性

### 5.3 Alpha 衰减与执行紧迫性

如果信号有半衰期（如 10 bar 后信号强度减半），则产生"执行紧迫性"：
- 信号快速衰减 → 必须快执行，牺牲冲击成本
- 信号缓慢衰减 → 可以慢慢执行，降低冲击

这导致目标函数中出现 **urgency term**：

```
total_cost = execution_cost + alpha_decay_cost
```

> **本项目是单期近视优化**，不建模 alpha 衰减和多期分拆。
> 当单笔交易量占 ADV > 10% 时，应考虑引入多期框架。

## 六、鲁棒优化

### 6.1 MVO 是 estimation error maximizer

Markowitz 优化器会在估计误差最大的方向上集中仓位——它把噪声当成信号。
经典实验：给 MVO 喂随机数据，它也能构建出"最优"组合（过拟合到噪声）。

### 6.2 不确定集方法

假设真实参数 `μ` 在集合 `U` 中（不确定集），求最坏情况下的最优：

```
maximize  min_{μ ∈ U}  μ'w - λw'Σw
```

- **椭球不确定集**：`U = {μ : (μ-μ̂)'Ω⁻¹(μ-μ̂) ≤ κ²}`
- 等价于在目标函数中添加 L2 正则项 `κ × ‖Ω^{1/2} w‖`
- 正则化强度 κ 控制保守程度

### 6.3 Black-Litterman 模型

另一种处理估计误差的方法：不从零开始估计 μ，而是从"市场均衡"出发，
用贝叶斯方法融入投资者的主观观点。

```
μ_BL = [(τΣ)⁻¹ + P'Ω⁻¹P]⁻¹ × [(τΣ)⁻¹π + P'Ω⁻¹q]
```

其中 π 是市场隐含均衡收益，P/q 是投资者观点，Ω 是观点不确定度。

优势：结果对输入参数不敏感，产生更稳定的权重。

### 6.4 正则化视角

从机器学习视角看：
- L2 正则（Ridge）↔ 椭球不确定集 → 权重分散
- L1 正则（Lasso）↔ 菱形不确定集 → 权重稀疏
- 换手惩罚 `γ‖Δw‖₁` 也是一种正则化 → 减少不必要调仓

> **本项目**通过动态成本函数隐式实现正则化（交易成本惩罚抑制频繁调仓），
> 但未显式使用鲁棒优化或 Black-Litterman。这是未来可扩展的方向。

## 七、求解器与工程实践

### 7.1 问题类型层级

| 问题类型 | 描述 | 常用求解器 |
|---------|------|-----------|
| LP | 线性目标 + 线性约束 | Simplex, Interior Point |
| QP | 二次目标 + 线性约束 | OSQP, Gurobi |
| SOCP | 二阶锥约束 | ECOS, MOSEK |
| SDP | 半定规划 | SCS, MOSEK |
| MIQP | 混合整数二次规划 | Gurobi, CPLEX |

标准 MVO（`w'Σw - λμ'w`）是 QP，可用 OSQP 高效求解。
加入 `|Δw|^1.5` 项后升级为 SOCP（二阶锥规划），需要 ECOS 或 MOSEK。

> **本项目**：Phase 2b 的 `PortfolioConstructor` 是 QP（OSQP），
> Phase 2c 的 `ExecutionOptimizer` 包含 1.5 次幂项，是 SOCP（cvxpy 自动选择 ECOS）。

### 7.2 工程技巧

- **热启动（warm start）**：用上一步的解作为当前步的初始点，加速收敛。
  特别适合事件驱动场景，相邻步之间的最优解变化不大。
- **分解（decomposition）**：当标的数很大时，将大问题拆分为多个小问题。
- **近似**：用线性化近似非线性约束，或用松弛降低问题难度。

> **本项目**在 `prob.solve(warm_start=True)` 中使用了热启动。

## 八、加密市场特有考量

### 8.1 永续合约资金费率

永续合约没有到期日，用资金费率机制锚定现货价格：
- 多头溢价时，多头付费给空头（正费率），抑制多头
- 空头溢价时，空头付费给多头（负费率），抑制空头

资金费率是持仓的**持续成本/收益**，必须纳入目标函数。
BTC 永续合约的资金费率在牛市可达年化 30%+，忽略它会严重失真。

> **本项目**在 `ExecutionOptimizer` 的目标函数中包含 `funding_rate'w` 项，
> 调用方须按 bar 频率归一化传入。

### 8.2 极端波动率与非平稳流动性

- 加密市场的日波动率可达传统市场的 5-10 倍
- 流动性在重大事件（如 FTX 暴雷）时可能瞬间枯竭
- 协方差矩阵的估计窗口需要更短（20-60 天），以快速适应 regime change

### 8.3 7×24 交易

- 没有开盘/收盘，但有流动性周期（亚洲/欧洲/美洲时段）
- 周末流动性通常较差
- 资金费率每 8 小时结算一次

### 8.4 交易所间差异

- 不同交易所的 orderbook 深度差异大
- 跨所套利需要考虑提币延迟和手续费
- 单一交易所的 ADV 可能不代表全市场流动性
